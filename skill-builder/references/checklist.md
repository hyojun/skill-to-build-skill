# Final Review Checklist

Use this checklist to walk through the skill with the user before generating the final version. Keep it conversational — don't just read the list out loud.

---

## The Name

- [ ] Is it short and clear? (2-3 words is ideal)
- [ ] Is it all lowercase with dashes between words? (like `meeting-notes`, not `Meeting Notes`)
- [ ] Does it match what the skill does? (someone should be able to guess from the name)

## The Description

- [ ] Does it say **what the skill does** in one sentence?
- [ ] Does it say **when Claude should use it** — including phrases people might actually say?
- [ ] Is it specific enough? ("Helps with projects" is too vague — "Creates project boards with tasks and assignments" is better)
- [ ] Is it short? (Keep it under a paragraph — think of it as an elevator pitch)

## The Steps

- [ ] Are they in the right order? (Would it make sense to follow them top to bottom?)
- [ ] Is each step one clear action? (Not a big blob of multiple things)
- [ ] Does Claude know what "done" looks like? (What's the expected result?)
- [ ] Is there anything missing? (Walk through a real scenario mentally — does anything get skipped?)

## Handling Problems

- [ ] What if the user gives incomplete information? (Does Claude know to ask follow-up questions?)
- [ ] What if something goes wrong? (Is there a fallback or a helpful error message?)
- [ ] What if the user wants something slightly different? (Is there flexibility built in?)

## The Examples

- [ ] Are there at least 2 examples?
- [ ] Do they sound like real requests someone would actually make? (Not overly formal or perfect)
- [ ] Does each example show what the user says, what Claude does, and what the result is?
- [ ] Do the examples cover different variations? (Simple vs. complex, different use cases)

## Final Gut Check

- [ ] If you were a new user, would you understand what this skill does just from reading the description?
- [ ] If you followed the steps, would you get a good result?
- [ ] Is there any confusing language that could be simplified?
